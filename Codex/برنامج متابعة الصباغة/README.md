# Dyehouse Temporary Prototype

نسخة تشغيلية أولى مستقلة لتجربة منطق النظام قبل دمجه مع المشروع الأساسي.

## ما تم تنفيذه

- نموذج إضافة أمر صباغة جديد
- سجل حركات للإستلام والتسليم والهالك
- محرك حساب تلقائي لكل أمر:
  - المستلم
  - المسلّم
  - رصيد المخزن
  - المتبقي
  - نسبة الهالك
- تحديث تلقائي للحالة:
  - بانتظار الاستلام
  - قيد التشغيل
  - جاهز للتسليم
  - مكتمل
- لوحة تحكم مرتبطة ببيانات تجريبية حية داخل الواجهة
- بحث سريع داخل الأوامر

## منطق الحساب الحالي

- رصيد المخزن = المستلم - المسلّم - الهالك
- المتبقي للتسليم = الكمية المطلوبة - المسلّم
- نسبة الهالك = الهالك / الكمية المطلوبة × 100

## ملاحظات دمج لاحقة

عند ربط النموذج بالمشروع الأساسي، يفضل فصل النظام إلى:

1. جدول Orders
2. جدول Movements
3. خدمة Balance Engine
4. خدمة Status Engine
5. Dashboard Queries مشتقة من الحركات بدل تخزين القيم يدويًا

## مبدأ مهم

الهالك يؤثر على رصيد المخزن، لكنه لا يُنقص الكمية التي ما زال العميل ينتظر استلامها. لذلك تم فصل المتبقي للتسليم عن الهالك الداخلي.

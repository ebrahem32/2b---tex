## ملاحظات تشغيل

- Railway يستخدم `DB_PATH=/data/2btex.sqlite`.
- عند أول تشغيل فقط، يتم نسخ `backend/data/2btex.sqlite` إلى Volume إذا لم تكن قاعدة `/data` موجودة.
- بعد أي تعديل في قاعدة seed داخل GitHub، قاعدة Railway الموجودة في Volume لن تتغير تلقائيًا إذا كانت `/data/2btex.sqlite` موجودة بالفعل.
- تحديث بيانات Railway يجب أن يتم عبر migration آمن أو إعادة تهيئة Volume بحذر شديد بعد Backup.

## مراجعة المخاطر

- قاعدة SQLite داخل GitHub مفيدة كنسخة تشغيلية مبدئية، لكنها ليست أفضل حل طويل الأجل للبيانات اليومية.
- بعد الانتقال إلى الـ Backend، يجب عدم الاعتماد على LocalStorage كمصدر أساسي.
- أي تعديل في الحسابات يجب اختباره عبر `/api/dashboard/summary` و `/api/bootstrap`.
